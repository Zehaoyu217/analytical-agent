# Gardener Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Source spec:** `docs/superpowers/specs/2026-04-19-gardener-design.md`
**Goal:** Ship the Gardener as a fourth pipeline phase with Tend button, 7 LLM passes, three-tier model selection, budget controls, dual-layer audit, and a configurable proposal/autonomous authority mode.

**Architecture:** New `second_brain/gardener/` package with `runner.py` + 7 pass modules. New FastAPI router `sb_gardener.py` exposing status/run/habits/estimate/log. Pipeline ledger extended with a `gardener` phase slot. Frontend adds a Tend button + drawer with model tiers / pass toggles / budget controls.

**Tech Stack:** Python 3.12 + FastAPI + Anthropic SDK (for claude-*) / OpenAI-compatible client (for OpenRouter) · React + Zustand + Vite.

---

## Scope of this plan

The spec has 7 build phases. This plan covers:

- **Phase 1** — Scaffolding (protocol, Proposal, audit, cost, budget, habits schema)
- **Phase 2** — Runner + `extract` pass end-to-end (Tend button works, claims land in `pending.jsonl`)
- **Phase 3** — `re_abstract` + `semantic_link` passes
- **Phase 4** — Settings UI (model tiers, pass toggles, budget, authority, dry-run)
- **Phase 5** — Activity panel + HUD phase dot
- **Phase 6** — Remaining passes (`dedupe`, `contradict`, `taxonomy_curate`, `wiki_summarize`)
- **Phase 7** — `autonomous` mode direct writes + `supersedes:` frontmatter

We ship Phase 1 + Phase 2 + Phase 4 (minimal) in this session so the user gets working LLM extraction with model selection. Phases 3–7 follow in a continuation session.

### Task 2.3 — Runner

- [ ] **Step 2.3.1** — Write `runner.py`:

```python
class GardenerRunner:
    def __init__(self, cfg: Config, habits: Habits, *, passes: list[GardenerPass] | None = None) -> None: ...
    def estimate(self) -> CostEstimate: ...  # calls pass.estimate() for enabled passes
    def run(self, *, dry_run: bool = False) -> RunResult: ...
```

- Behavior: builds `Budget` from habits; for each enabled pass (in fixed order), calls `pass.run(cfg, habits, client, budget)` yielding `Proposal`s; appends each to `pending.jsonl` (prefixed with pass origin) and `gardener.log.jsonl`; hard-stops on `BudgetExceeded`; returns `RunResult(passes_run, proposals_added, total_tokens, total_cost_usd, duration_ms, errors)`.
- Also writes the summary into the pipeline ledger via `sb_pipeline_state.write_phase(cfg, "gardener", summary)`.

- [ ] **Step 2.3.2** — Test `test_runner.py`:
  - With stubbed passes, assert proposals land in both `pending.jsonl` and `gardener.log.jsonl`
  - Dry-run emits no API calls and no writes
  - Budget-exceeded halts mid-run; partial proposals kept
